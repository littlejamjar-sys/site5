# Security Policy

## Reporting Security Vulnerabilities

If you discover a security vulnerability in site5, please report it responsibly by emailing security@littlejamjar-sys.dev instead of using the public issue tracker.

### Security Report Guidelines

Include the following information:
- Description of the vulnerability
- Steps to reproduce (if applicable)
- Potential impact
- Suggested fixes (if any)

We will:
- Acknowledge receipt within 48 hours
- Investigate and assess the severity
- Develop and release fixes as quickly as possible
- Credit you in the security advisory (unless you prefer anonymity)

## Security Measures

This project implements the following security practices:

- **Dependency Scanning**: Automated vulnerability checks on JavaScript dependencies
- **SBOM Generation**: Software Bill of Materials for supply chain transparency
- **Security Audits**: Regular automated security audits

## Supported Versions

Latest version is always supported and maintained with security updates.

## Contact

For security concerns, contact: security@littlejamjar-sys.dev
